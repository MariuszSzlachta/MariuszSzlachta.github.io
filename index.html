About me

My name is Mariusz. I'm an ambitious web developer with a creative approach for solving problems and a huge dose of sense of humor. Creating applications and websites gives me great satisfaction. I love the feeling when complex parts of code become clear and problems, previously impossible to solve, become easy to overwcome.

I'm a graduate of the Web Developer bootcamp organized by Kodilla, during which I built solid foundation in the area of web apps development based on the most popular Javascript technologies. In my projects I'm eager to use solutions offered by ReactJS and state management provided by Redux. SASS in SCSS syntax became my friend. Task automation using npm and building applications using a webpack is almost an indispensable standard in my work. It's g­reat that during the bootcamp I was able to get familiar with backend technologies such as MongoDB with Mongoose, Node.js and Express. However, the most valuable thing I've learned is that nothing is impossible.

I'm currently diving in React by doing a great React course on Udemy.I still create more apps that you can see on my GitHub but my main goal is to find a good job that will give me an opportunity to be involved in ambitious and interesting projects. I want to constantly develop my skills and learn from more experienced team members. I encourage you to familiarize yourself with the most interesting projects I have carried out.

Kanban

Project details

Kanban

Isomorphic app | July 2018

This is the bigges project I've made so far. My app handles CRUD operations for columns and notes, as well as provides a 'drag and drop' functionality for notes which is possible due to the react-html5-dnd library.

From a technical side, the application is as follows. On backend I've used Express.js to create a server. RESTful API is responsible for handling the CRUD operation on mongoDB database (hosted on MLab). Of course I’ve used a Mongoose to make a work with database more comfortable. To generate modules, components, models I've chosen a mern generator that uses EJS templates. A development environment is managed by Webpack and NPM scripts. On frontend views are generated by React while Redux manages a state of the app. Since it's an isomorphic application, its state is managed asynchronously by Redux-thunk. The whole project was based on the MERN-template.

The application has been placed on the Heroku website, therefore the first launch might take a while, because Heroku disables applications that are not currently in use.

Project details

Country App

Single Page App | Juni 2018

Country is a classic expample of Single Page Applications. It presents the basic information about the selected country in an interesting way.

The app was created with React and React Router libraries, and entire rendering lies with a client side. Information about countries are loaded from *.json file. A user can filter the list of countries depending on a chosen continent. Clicking the flag app enables viewing a proper component that presents detailed information about a chosen country. There are also buttons for removing a country from the list. By using the contact component, you can go to my portfolio or send me an e-mail. You can switch between main components using navbar on top.

The process of creation of this application has helped me to understand the philosophy of SPA, as well as to practice the most popular frontend javascript libraries.
